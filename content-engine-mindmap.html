cofenster Content Engine — Workflow Map

Full autonomous pipeline: research → write → image → publish → social

What is this?

The Content Engine is an AI-powered autonomous agent that runs inside Claude (Cowork). Its job is simple: make cofenster look alive from the outside. Marketing needs to publish 2–3 blog posts per week plus LinkedIn content, but doesn't have a dedicated content writer to sustain that cadence. This agent replaces that role.

Every time it runs, it scans the competitive landscape and industry news (across 165+ sources including competitor blogs, HR tech publications, and internal comms outlets), picks the single best topic for cofenster's audience, and produces two ready-to-publish deliverables: a bilingual blog post (English + German) pushed directly to the cofenster Webflow CMS as drafts, and a LinkedIn carousel with post text.

All content is SEO-optimized (keywords, meta tags, structured headings) and AEO-optimized — meaning it includes FAQ sections and answer-style formatting designed to surface in LLM-generated answers and AI search results like ChatGPT, Perplexity, and Google AI Overviews. The goal is to rank in both traditional Google search and in the new wave of AI answer engines.

Tom reviews and publishes — the agent never auto-publishes. It's designed to run daily on a schedule, producing content autonomously and notifying Tom when it's done.

How it flows

The agent works as a linear pipeline with 6 phases. It researches by scanning web sources, HubSpot deals, Slack, and Google Drive to find the best topic and avoid duplicates. Then it writes an English blog first, followed by a culturally adapted German version (not a translation) — both following cofenster brand guidelines. It generates a hero image using Google's Imagen 4.0 model. It pushes both blog versions to Webflow as CMS drafts with full SEO meta tags, OG fields, and FAQ slots. It creates a 6-slide LinkedIn carousel as a self-contained HTML file (using cofenster brand visuals) plus ready-to-paste post text. Finally, it delivers everything — 6 output files + Webflow draft IDs — and reports back. Each phase feeds the next; if one breaks (e.g. Webflow is down), the pipeline continues and delivers what it can.

Important Constraints
⚠ Limitations
Always drafts — isDraft: true, never auto-publish. Tom reviews first.

No image upload — Webflow asset_tool can't upload from disk. Thumbnail left empty for manual upload.

CTA links — Link-type fields may reject strings. CTA is embedded in blog body as fallback.

Exactly 3 FAQs — CMS has 3 FAQ slots. No more, no fewer.
⚡ Edge Case: Webflow connection fails
Still produces blog markdown files. Tom can manually paste into Webflow. API issues don't block the pipeline.
